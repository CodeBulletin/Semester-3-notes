Q9

(a) What is reducibility? State the necessary conditions for reducibility. Explain reducibility with an example. 

Reducibility in computational complexity theory refers to the method of comparing the complexity of two problems by showing that one (the target problem) can be transformed or "reduced" to the other (the source problem). This concept is fundamental in classifying problems, particularly in demonstrating NP-completeness.

Necessary Conditions for Reducibility

Polynomial Time Transformation: The reduction from problem A (source) to problem B (target) must be achievable in polynomial time. This means there exists an algorithm that can transform any instance of problem A into an instance of problem B in polynomial time.

Solution Preservation: The transformation must preserve solutions, meaning that solving the transformed instance of problem B should give information about the solution to the original instance of problem A.

Example of Reducibility

Consider the problem of the Boolean Satisfiability Problem (SAT) and the problem of 3-SAT (a specific version of SAT where each clause has exactly three literals).

  • SAT: Given a Boolean formula, determine if there is an assignment of truth values to variables that makes the entire formula true.
  • 3-SAT: A special case of SAT where the formula is in conjunctive normal form (CNF) and each clause has exactly three literals.

Reduction from SAT to 3-SAT:

  • We can transform any instance of SAT into an instance of 3-SAT. This involves rewriting the clauses in the SAT formula so that each clause has exactly three literals, using logical equivalences and additional variables if necessary.
  • This transformation can be done in polynomial time, and a solution to the 3-SAT instance directly gives a solution to the original SAT instance.

(b) Explain the circuit satisfiability problem and show that it is an NP Complete problem.

Circuit Satisfiability Problem (Circuit-SAT)

  • Definition: Given a combinational logic circuit (built from AND, OR, NOT, etc., gates) with a single output and several inputs, the Circuit-SAT problem asks whether there is a combination of inputs that results in the output being true (1).

Showing Circuit-SAT is NP-Complete

Circuit-SAT is in NP:

  • To prove that Circuit-SAT is in NP, we need to show that if we are given a "certificate" (a specific input assignment), we can verify in polynomial time whether this assignment makes the circuit output true. This is straightforward, as evaluating a logic circuit with a given input is a polynomial-time task.

Reduction from a Known NP-Complete Problem:

  • The usual approach to show that a problem is NP-Complete is by reducing a known NP-Complete problem to it.
  • For Circuit-SAT, a common strategy is to reduce from 3-SAT (known to be NP-Complete).
  • Each clause in a 3-SAT formula can be represented as a small circuit with three inputs (the literals of the clause) and one output. These small circuits can be combined in a way that the entire 3-SAT formula is represented by a larger circuit.
  • This reduction can be done in polynomial time, and a 3-SAT formula is satisfiable if and only if the corresponding circuit is satisfiable.

Through this reduction, we've shown that Circuit-SAT is at least as hard as 3-SAT, a known NP-Complete problem. Hence, Circuit-SAT is NP-Complete.
