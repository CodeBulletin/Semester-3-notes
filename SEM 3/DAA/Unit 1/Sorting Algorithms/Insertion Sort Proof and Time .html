Insertion Sort Proof and Time Complexity

Defining the Loop Invariant

For Insertion Sort, the loop invariant we consider is:

"At the start of each iteration of the main loop (say, the i-th iteration), the first i elements of the array are in sorted order."

Initialization

Before the first iteration of the loop (i.e., when i = 1), the invariant holds because the first element (or the first 0 elements if we consider an empty sublist as sorted) is trivially sorted.

Maintenance

Assuming the invariant is true before an iteration of the loop, we need to show it remains true after the iteration. During the i-th iteration, the algorithm places the i-th element into its correct position in the first i elements, ensuring that now the first i+1 elements are sorted. This maintains the invariant.

Termination

At the end of the algorithm, when the loop completes, the invariant tells us that the first n elements are sorted, where n is the total number of elements in the array. This means the entire array is sorted.

Conclusion

The maintenance of the loop invariant, along with its initialization and termination conditions, prove that the Insertion Sort algorithm correctly sorts the list. Each iteration of the loop extends the sorted section of the array by one element, ensuring that the array is fully sorted when the algorithm completes.
