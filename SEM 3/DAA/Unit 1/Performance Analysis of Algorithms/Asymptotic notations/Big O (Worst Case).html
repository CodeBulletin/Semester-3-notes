Big O (Worst Case)

Big O notation describes the upper bound or worst-case behavior of a function or algorithm. It provides an upper limit on the growth rate of a function. In other words, it characterizes how a function or algorithm behaves when the input size becomes very large.

For a function \(f(n)\),  \(O(g(n))\) represents an upper bound on its growth. This means that there exists a constant \(c > 0\) and an input size \(n_0\) such that  \(f(n)\) is bounded above by \(c *g(n) \space \forall \space n \ge n_0\)

Example: If an algorithm's time complexity is O(n^2), it means that the algorithm's running time grows no faster than the square of the input size.
