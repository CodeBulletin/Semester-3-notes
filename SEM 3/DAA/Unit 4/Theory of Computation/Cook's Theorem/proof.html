Cook's Theorem, named after the mathematician Stephen Cook, is a fundamental theorem in the field of computational complexity theory. Formulated in 1971, this theorem established the concept of NP-completeness, which has since been a central topic in computer science. Here's a summary of what Cook's Theorem states and its significance:

Statement of Cook's Theorem

Cook's Theorem asserts that the Boolean satisfiability problem (often referred to as SAT) is NP-complete. In other words, SAT is at least as hard as the hardest problems in NP, and it is itself a problem in NP. The formal statement of Cook's Theorem is:

"The SAT problem is NP-complete."

This means two things:

  1. SAT is in NP: A solution to a SAT problem (a truth assignment to variables) can be verified quickly (in polynomial time).
  2. SAT is NP-hard: Any problem in NP can be reduced to SAT in polynomial time.

Proving Cook's Theorem

Proving Cook's Theorem involves showing that the Boolean Satisfiability Problem (SAT) is NP-complete. The proof is technical and involves two main parts:

Proving SAT is in NP: This is straightforward. Given a Boolean formula and an assignment of truth values to variables, we can verify in polynomial time whether the formula is satisfied (i.e., evaluates to true).

Proving SAT is NP-hard: This is the crux of the proof. It involves showing that any problem in NP can be reduced to SAT. This is done by simulating the computation of a nondeterministic Turing machine (which defines NP problems) using a Boolean formula.

Here's a high-level overview of how the NP-hardness proof works:

Constructing the SAT Instance from a General NP Problem

Nondeterministic Turing Machine (NTM): Consider an NTM that decides a problem in NP. Without loss of generality, assume this NTM runs in polynomial time with respect to the input size.

Simulation with Boolean Formula:

  • Encoding the Computation: We encode the entire computation of the NTM as a Boolean formula. This includes the state of the machine, the symbols on the tape, and the head position at each step.
  • Time and Space Polynomial: Since the NTM runs in polynomial time, the number of steps in the computation is polynomial in the size of the input. The tape used by the machine is also polynomially bounded.
  • Variables and Clauses: For each tape cell, head position, and time step, we create Boolean variables. We then construct clauses that represent the transition function of the NTM. These clauses ensure that the variables correctly simulate the behavior of the NTM.
  • Initial and Final Conditions: Additional clauses represent the initial state (input) of the machine and the accepting condition.

Satisfiability Implies Solution:

  • If the resulting Boolean formula is satisfiable, it encodes a valid computation of the NTM that leads to an accepting state. This means the original problem instance is a "yes" instance.
  • Conversely, if the NTM accepts the input, there exists a sequence of transitions leading to an accepting state, which can be encoded as a satisfying assignment to the Boolean formula.

Conclusion

The transformation from any NP problem to a SAT instance is done in polynomial time (since both the number of variables and the size of the formula are polynomial in the size of the NTM's input). Hence, any problem in NP can be reduced to SAT, proving that SAT is NP-hard.

Since SAT is both in NP and NP-hard, it is NP-complete, which is the essence of Cook's Theorem.

This proof was groundbreaking because it provided the first concrete example of an NP-complete problem and established a method for proving the NP-completeness of other problems.
