TSP proof

To prove that the Traveling Salesman Problem (TSP) is NP-complete, we use a reduction from the Hamiltonian Circuit Problem, which is known to be NP-complete. The Hamiltonian Circuit Problem involves finding a Hamiltonian circuit (a path in a graph that visits each vertex exactly once and returns to the starting vertex) in a given graph.

The process involves two main steps:

Show that TSP is in NP: This means demonstrating that any proposed solution (a tour in this case) to the TSP can be verified quickly (in polynomial time).

Show that TSP is NP-hard: This involves proving that any problem in NP, specifically the Hamiltonian Circuit Problem in this case, can be reduced to TSP in polynomial time.

1. TSP is in NP

To show that TSP is in NP, we need to demonstrate that given a particular tour, we can verify whether this tour is a valid solution (visits each city exactly once and returns to the starting city) and whether the total distance of the tour is less than or equal to a given value (for decision-version TSP) in polynomial time. Verifying a tour involves checking the order of cities and summing the distances, which can be done in polynomial time. Therefore, TSP is in NP.

2. TSP is NP-hard

To prove NP-hardness, we use a reduction from the Hamiltonian Circuit Problem. The goal is to show that any instance of the Hamiltonian Circuit Problem can be transformed into an instance of TSP in such a way that the graph has a Hamiltonian circuit if and only if there is a tour in the TSP instance with a total distance less than or equal to a specific value.

Here's how the reduction works:

Step 1: Take an instance of the Hamiltonian Circuit Problem, represented by a graph \(G\).

Step 2: Create an instance of TSP using the same graph  \(G\). Set the distance between two cities (vertices) to 1 if there is an edge between them in \(G\), and to a very large number (say \(M\), larger than the number of vertices in \(G\)) if there is no edge.

Step 3: Set the maximum allowed distance for the TSP tour to be equal to the number of vertices in \(G\).

Now, if \(G\) has a Hamiltonian circuit, there will be a TSP tour in the corresponding TSP instance with a total distance equal to the number of vertices in \(G\) (since each edge in the Hamiltonian circuit contributes a distance of 1). Conversely, if there is a TSP tour with a total distance equal to the number of vertices in \(G\), it means that the tour only uses edges that exist in \(G\), and since it visits each vertex exactly once before returning to the start, it forms a Hamiltonian circuit in \(G\).

This transformation can be done in polynomial time. Since we've shown that any instance of the Hamiltonian Circuit Problem can be reduced to TSP in polynomial time, and verifying a TSP solution is in NP, it follows that TSP is NP-complete. This proof has significant implications in computational complexity theory and practical applications involving optimization and routing problems.
