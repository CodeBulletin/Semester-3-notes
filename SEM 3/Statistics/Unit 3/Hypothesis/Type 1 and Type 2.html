Type 1 and Type 2

In hypothesis testing and statistical analysis, Type I and Type II errors are two different types of mistakes that can occur when making decisions based on data and hypothesis tests. These errors are associated with the acceptance or rejection of the null hypothesis (H0) in the context of a statistical test.

  1. Type I Error (False Positive, α):
    • Type I error occurs when the null hypothesis (H0) is incorrectly rejected when it is, in fact, true.
    • In other words, it's a false positive or a "false alarm."
    • This error is also known as the significance level (α), and it represents the probability of making a Type I error.
    • In statistical terms, it means that you conclude there is a significant effect, relationship, or difference when there isn't one.
    • The probability of committing a Type I error is typically set as the significance level (α) and is controlled by the researcher. Common values for α are 0.05 (5%) or 0.01 (1%).

Example: Concluding that a new drug is effective when, in reality, it has no effect.

  1. Type II Error (False Negative, β):
    • Type II error occurs when the null hypothesis (H0) is incorrectly not rejected when it is, in fact, false.
    • It's a false negative or a "missed opportunity."
    • This error is also represented by the probability β (beta).
    • In statistical terms, it means that you fail to detect a significant effect, relationship, or difference when it actually exists.
    • The probability of committing a Type II error depends on factors such as sample size, the effect size, and the significance level (α).

Example: Failing to conclude that a new drug is effective when it does have a positive effect.
