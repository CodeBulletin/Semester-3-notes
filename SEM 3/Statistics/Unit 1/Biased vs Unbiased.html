Biased vs Unbiased

Unbiased Estimation: An estimator is called unbiased if, on average, it produces the true value of the parameter being estimated. This means that if the estimation process was repeated many times, the average of the estimates would be equal to the true parameter value. An unbiased estimator does not systematically overestimate or underestimate the parameter.

Mathematically, an estimator \(\^θ\) for a parameter \(θ\) is unbiased if the expected value of \(\^θ\) is equal to 
\(E(\^\theta) = \theta\)

For example, the sample mean is an unbiased estimator of the population mean. If you take many samples from a population and calculate the mean of each sample, the average of all these sample means will be very close to the population mean.

Biased Estimation: A biased estimator, on the other hand, does not have this property. If an estimator is biased, it means that its expected value is not equal to the true parameter value. In other words, a biased estimator systematically overestimates or underestimates the parameter.

Mathematically, an estimator \(\^θ\) is biased if:

\(E(\^\theta) \ne \theta\)

The bias of an estimator is the difference between the estimator's expected value and the true value of the parameter: \(\text{Bias}(\^\theta) =  E(\^\theta) - \theta\)

For example, if you use the sample variance formula without applying Bessel's correction (i.e., dividing by \(n\) instead of \(n-1\) for a sample size  \(n\)), you get a biased estimator of the population variance. This is because it tends to underestimate the variance, especially for small sample sizes.
