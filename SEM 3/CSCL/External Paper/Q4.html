(a) What are Cyber Security Vulnerabilities and what are the common types of Cyber Security Vulnerabilities?

What are Cyber Security Vulnerabilities?

Cyber security vulnerabilities refer to weaknesses or flaws in a system's design, implementation, operation, or management that can be exploited by cybercriminals to gain unauthorized access, cause damage, or disrupt operations.

Common Types of Cyber Security Vulnerabilities

  1. Software Flaws: Bugs or errors in software that can be exploited, like buffer overflows or unvalidated input.
  2. Unpatched Software: Software that hasn’t been updated with the latest security patches, leaving known vulnerabilities open to exploitation.
  3. Weak Authentication: Poorly implemented authentication mechanisms that make it easy to intercept or guess user credentials.
  4. Misconfigurations: Incorrect system or security settings that leave systems vulnerable to attack.
  5. Insecure Interfaces and APIs: Poorly designed or unprotected application programming interfaces (APIs) that can be exploited to gain unauthorized access or information.
  6. Outdated or Weak Encryption: Using obsolete or weak cryptographic standards that can be easily broken.
  7. Phishing and Social Engineering: Tactics used to deceive individuals into divulging confidential or personal information that may be used for fraudulent purposes.
  8. Insider Threats: Threats from individuals within the organization who may have malicious intent or inadvertently cause harm due to negligence.
  9. Physical Security Breaches: Failure to secure physical access to critical hardware and infrastructure.

(b) How to Prevent an SQL Injection and How is an SQL Injection attack performed?

How to Prevent SQL Injection

  1. Use Prepared Statements: Using parameterized queries or prepared statements with placeholders for data ensures that the input is treated as data and not as part of the SQL command.
  2. Sanitize Input: Validate and sanitize user input to ensure it doesn’t contain malicious SQL.
  3. Use Stored Procedures: They can encapsulate the SQL logic and prevent exposure of underlying SQL syntax.
  4. Least Privilege: Ensure that database accounts used by applications have the least privilege necessary, limiting the data they can access and modify.
  5. Regularly Update and Patch: Keep software and dependencies up-to-date with the latest security patches.
  6. Web Application Firewalls (WAFs): Deploy WAFs to detect and block SQL injection attempts.
  7. Error Handling: Implement proper error handling that doesn’t expose sensitive information in error messages.

How SQL Injection Attacks are Performed

  1. Exploiting Input Fields: Attackers input malicious SQL code into fields like login forms, search boxes, or URL parameters where user input is expected.
  2. Manipulating SQL Queries: The attacker's input is designed to alter the SQL query being executed by the application. For example, appending a OR 1=1 statement to a login query to bypass authentication.
  3. Exfiltrating Data: Successful SQL injection can allow attackers to view, modify, or delete data in the database, depending on their complexity and the database’s configuration.
  4. Elevating Privileges: Advanced SQL injection techniques can be used to escalate privileges and perform administrative operations on the database.

SQL injection attacks exploit vulnerabilities in web application software when user input is not correctly checked or sanitized before being incorporated into SQL statements.
