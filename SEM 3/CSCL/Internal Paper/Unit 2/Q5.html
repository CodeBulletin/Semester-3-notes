(a) What is SQL injection? How is it executed? How Can we protect data from this attack

What is SQL Injection?

SQL Injection is a code injection technique that exploits vulnerabilities in an application's software by injecting malicious SQL statements into an entry field for execution. This can allow attackers to access, modify, or delete data, or even gain administrative access to a database.

How is It Executed?

  1. Input Manipulation: Attackers input malicious SQL code into vulnerable input fields (like login forms) where inputs are not properly sanitized.
  2. Exploiting Vulnerabilities: The application then passes this malicious input to the database, which executes the SQL command, compromising the database's security.

Protection Measures

  1. Input Validation: Ensure that all user inputs are validated for type, length, format, and range.
  2. Prepared Statements and Parameterized Queries: Use these in the application code to separate SQL code from data.
  3. Stored Procedures: Employ stored procedures to encapsulate the SQL logic.
  4. Regular Security Audits: Regularly audit and test the applications for SQL injection vulnerabilities.
  5. Least Privilege: Assign the minimum necessary privileges to database accounts.
  6. Web Application Firewalls (WAF): Use WAFs to detect and block SQL injection attacks.

(b) What are backdoors, and how do they have the potential to compromise the security and integrity of our system?

What are Backdoors?

Backdoors are hidden entry points into a software, network, or system that bypass normal authentication and security mechanisms. They are often used for legitimate purposes, like providing system maintenance, but can be exploited for malicious activities.

Potential Compromises

  1. Unauthorized Access: Backdoors can allow attackers to gain unauthorized access to a system.
  2. Data Theft and Manipulation: Once access is gained, data can be stolen, manipulated, or destroyed.
  3. Surveillance: Attackers might use backdoors for continuous monitoring of activities on the system.
  4. Spreading Malware: They can be used to spread malware within a network.

(c) Elaborate DoS aftack. Classify various DoS attacks and Explain Flood attack, ping of death and SYN attack.

Elaboration on DoS Attack

DoS attacks aim to make a machine or network resource unavailable to its intended users, typically by temporarily or indefinitely disrupting services of a host connected to the Internet.

Classification and Explanation

Flood Attack:

  • Mechanism: Overwhelming the target with excessive requests, making it impossible for the server to respond to legitimate traffic.
  • Example: UDP flood attack.

Ping of Death:

  • Mechanism: Involves sending malformed or oversized packets using the ICMP protocol, causing systems to crash or reboot.
  • Vulnerability: Older and unpatched systems are particularly susceptible to this attack.

SYN Attack:

  • Mechanism: The attacker sends a flood of SYN requests to the target's system in an attempt to overwhelm it, making it unable to respond to legitimate connection requests.
  • Impact: Consumes server resources or fills the connection table, leading to denial of service.

Prevention and mitigation of DoS attacks typically involve network security measures like firewalls, intrusion detection systems, and traffic filtering.
