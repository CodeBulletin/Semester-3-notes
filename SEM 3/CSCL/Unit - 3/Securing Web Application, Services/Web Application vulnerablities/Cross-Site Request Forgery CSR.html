Cross-Site Request Forgery CSRF

Cross-Site Request Forgery (CSRF) is a type of security attack on web applications. It tricks the user into performing unwanted actions on a web application where they are currently authenticated. CSRF attacks specifically target state-changing requests, not theft of data, since the attacker has no way to see the response to the forged request.

How CSRF Works:

User Authentication: The user logs into a web application, and the application authenticates the user.

Session Cookie: The application creates a session cookie that is stored in the user's browser. This cookie is used for subsequent requests to authenticate the user.

Malicious Request: The attacker tricks the user into making an unwanted request to the web application (e.g., through a deceptive email, link, or a form on a different website). This request is made with the user's credentials (like cookies) automatically included by the browser.

Execution of Request: The web application can't distinguish this malicious request from a legitimate request made by the user and executes it.
