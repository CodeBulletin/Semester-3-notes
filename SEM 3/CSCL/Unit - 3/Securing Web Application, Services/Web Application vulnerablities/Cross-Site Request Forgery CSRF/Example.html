Example

Example of a CSRF Attack:

Imagine a web application where users can change their email address via a form that sends a POST request. A typical request might look like this:

POST /changeEmail
Host: example.com
Content-Type: application/x-www-form-urlencoded
Cookie: sessionid=abc123

email=newemail@example.com

An attacker can craft a malicious website with a form like this:

<form action="http://example.com/changeEmail" method="POST">
  <input type="hidden" name="email" value="attacker@example.com" />
</form>
<script>document.forms[0].submit();</script>

If a user who is logged into example.com visits this malicious site, the script automatically submits the form, sending a request to example.com to change the user's email to the attacker's email. Since the user's browser automatically includes the session cookie for example.com, the application considers it a legitimate request.
