Example of SQL Injection:

Suppose a website has a login form where users enter their username and password. The website checks the credentials by running an SQL query:

SELECT * FROM users WHERE username = '[username]' AND password = '[password]';

Here, [username] and [password] are placeholders for the user-supplied input. An attacker can exploit this by entering a username that includes an SQL fragment:

Input:

  • Username: admin' --
  • Password: (any or blank)

This changes the query to:

SELECT * FROM users WHERE username = 'admin' --' AND password = '';

The -- sequence is a comment in SQL, so everything after the admin' is ignored. If the admin user exists, the attacker gains access without needing a password.
